  <action>
Create a multi-stage Dockerfile at the repository root. This is a minimal Dockerfile focused on closing the OCR-02 gap — Phase 5 (Containerization, CONT-01) will refine it further.

**Stage 1: builder**
- Base image: `python:3.11-slim` (matches project Python version)
- Install build dependencies needed for pip compilation: `build-essential`, `libpq-dev`
- Copy `requirements-prod.txt` and run `pip install --no-cache-dir -r requirements-prod.txt`
- Use `--prefix=/install` to isolate installed packages for copying

**Stage 2: runtime**
- Base image: `python:3.11-slim`
- Install runtime system packages with `apt-get update && apt-get install -y --no-install-recommends tesseract-ocr tesseract-ocr-eng libpq5 && rm -rf /var/lib/apt/lists/*`
  - `tesseract-ocr`: the OCR binary that pytesseract shells out to
  - `tesseract-ocr-eng`: English language data (per CONTEXT.md decision: English only to keep image smaller)
  - `libpq5`: PostgreSQL client library needed by asyncpg/psycopg at runtime
- Copy installed Python packages from builder stage: `COPY --from=builder /install /usr/local`
- Copy application code: `COPY . /app`
- Set `WORKDIR /app`
- Set `ENV PYTHONUNBUFFERED=1`
- Expose port 8000
- Default CMD: `["uvicorn", "main:app", "--host", "0.0.0.0", "--port", "8000"]`
- Add a comment noting the Celery worker alternative: `# Celery worker: celery -A celery_app worker --concurrency=1 --pool=solo`

Do NOT add HEALTHCHECK, USER, or ENV for secrets — those refinements belong to Phase 5 (CONT-01).
