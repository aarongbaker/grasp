## Trust Boundaries

- Pinecone metadata crosses a network boundary and must not be trusted solely because the query included an ownership filter.
- The enricher’s per-recipe lookup path should avoid duplicate remote queries inside one pipeline run while preserving graceful degradation.

## STRIDE Register

| threat_id | category | component | disposition | mitigation_plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-05 | Information Disclosure | `_retrieve_rag_context()` returned metadata | mitigate | Assert returned ownership metadata matches the requesting user, log mismatches, and drop those chunks before enrichment uses them. |
| T-03-06 | Denial of Service | repeated enricher RAG retrievals | mitigate | Add a session/query-scoped per-run cache so equivalent lookups reuse the same retrieval result instead of repeatedly querying Pinecone. |
</threat_model>
