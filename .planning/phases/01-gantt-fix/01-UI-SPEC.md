## Interaction Contract

### Hover
- `.barGroup:hover` raises z-index to 2 (already implemented — retain).
- Browser-native `title` attribute on `.barGroup` shows full action text on hover. No custom tooltip — locked decision from CONTEXT.md.
- No hover color change on bars.

### Scroll
- The chart container must support horizontal scroll for sessions longer than the viewport width.
- Fixed chart width — do not scale-to-fit. Locked decision from CONTEXT.md.
- Implementation: wrap `.container` interior (`.timeAxis` + `.chart`) in a `overflow-x: auto` scroll container. Apply `min-width` based on total duration at a fixed px-per-minute rate.
- Smooth scroll behavior: `scroll-behavior: smooth` on the scroll container.
- `prefers-reduced-motion` override: remove smooth scroll when motion is reduced.

### Focus / Keyboard
- `.barGroup` elements must be keyboard-reachable: add `tabIndex={0}` to each `.barGroup`.
- On focus, apply copper focus ring: `outline: 2px solid var(--accent-primary); outline-offset: 2px`.
- Focus must be visually distinct from hover (hover = z-index lift only; focus = copper ring).

### Clock time fallback
- When `clock_time` is non-null on `TimelineEntry` objects: display absolute times ("4:00 PM", "4:30 PM") on x-axis. Locked decision from CONTEXT.md.
- When `clock_time` is null (no serving time set): fall back to relative offset labels ("+0m", "+30m", "+1h"). Locked decision from CONTEXT.md.
- The fallback path uses the existing `formatOffset()` helper — retain as-is.
- 12-hour format with AM/PM. Locked decision from CONTEXT.md.

## Component States

### CookingGantt states

| State | Trigger | Visual Treatment |
|-------|---------|-----------------|
| Normal render | Day-of steps present | Lane rows with positioned bars |
| No data | `lanes.length === 0` | Returns `null` (component does not render) — retain existing behavior |
| Clock time available | `clock_time` non-null on entries | X-axis shows "4:00 PM" format labels |
| Clock time absent | `clock_time` null on all entries | X-axis shows "+0m", "+30m" relative labels |
| Buffer present | `buffer_minutes > 0` on a step | Bar has solid color segment + 0.3 opacity extension for buffer zone |
| Very short step | `duration_minutes` results in bar narrower than 8px minimum | Bar renders at 8px minimum width; label hidden if bar narrower than 24px |

### Time marker interval logic (Claude's Discretion — resolved here)
- Total duration <= 90 minutes: markers every 15 minutes.
- Total duration > 90 minutes and <= 240 minutes: markers every 30 minutes.
- Total duration > 240 minutes: markers every 60 minutes.

### Minimum bar width (Claude's Discretion — resolved here)
- Minimum rendered bar width: 8px (up from current 4px `min-width`).
- Bar label hidden when the bar's rendered width is less than 24px (label would be unreadable).
