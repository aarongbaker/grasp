## Trust Boundaries

- `finalise_session()` owns terminal session writes and must not lose the `CANCELLED` terminal state.
- The cancellation route is user-triggered and races a Celery worker writing terminal results.

## STRIDE Register

| threat_id | category | component | disposition | mitigation_plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | `app/core/status.py` terminal write path | mitigate | Replace `db.get()`/`refresh()` with a single `select(Session)...with_for_update()` read and assert the locked row remains `CANCELLED` when already cancelled. |
| T-02-02 | Repudiation | `cancel_pipeline()` route contract | mitigate | Lock the session row before writing `CANCELLED`, keep terminal-state no-op behavior, and cover the HTTP path with targeted tests. |
</threat_model>
