<threat_model>
## Trust Boundaries

- OpenAI embedding calls cross a network boundary and must not leak client resources across Celery jobs.
- Partial failures in fallback embedding must degrade by skipping failed chunks, not by aborting successful ingestion work.

## STRIDE Register

| threat_id | category | component | disposition | mitigation_plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-03 | Denial of Service | `app/ingestion/embedder.py` OpenAI client lifecycle | mitigate | Use one `async with AsyncOpenAI(...) as openai_client:` at function entry so Celery workers do not accumulate open clients across ingestion calls. |
| T-02-04 | Tampering | fallback embedding result set | mitigate | Use `asyncio.gather(return_exceptions=True)` with `asyncio.Semaphore(10)` and post-filter exceptions to `None` so failed chunks are logged and skipped without aborting successful vectors. |
</threat_model>
