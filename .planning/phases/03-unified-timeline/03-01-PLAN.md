# Plan 03-01: Unify Timeline

## Objective

Merge the separate "Prep Ahead" and "Day-of Recipe Steps" sections into a single chronological timeline. Steps that can be done ahead get an inline tag instead of being pulled into a separate section. The Gantt chart shows all steps, making gaps and parallel tasks visible.

## Tasks

### Task 1: Unify the backend renderer

**File:** `graph/nodes/renderer.py`

1. Change `_build_timeline()` to return a single `list[TimelineEntry]` instead of a `tuple[list, list]`:
   - Remove the split loop that separates day_of from prep_ahead
   - Remove the `label = "Prep"` override for prep-ahead entries
   - Keep `is_prep_ahead` flag on entries (data integrity) but don't use it for splitting
   - Return all entries in one sorted list

2. Update `schedule_renderer_node()`:
   - `timeline = _build_timeline(merged_dag, serving_time)` (single list)
   - Set `prep_ahead_entries=[]` on NaturalLanguageSchedule (backwards compat)
   - Update both the success and error paths

3. Update `_format_schedule_for_prompt()` — remove the `[prep-ahead: ...]` suffix since steps aren't separated anymore. Instead, add a subtle `[can do ahead: {window}]` note.

**Commit:** `fix(03-01): unify renderer timeline — stop splitting prep-ahead`

### Task 2: Unify the frontend components

**File:** `frontend/src/components/session/ScheduleTimeline.tsx`

1. Remove the `prepAhead`/`mainTimeline` split logic (lines 92-97)
2. Remove the entire "Prep Ahead" section (lines 121-130)
3. Remove the `PrepItem` component (lines 42-61)
4. Rename "Day-of Recipe Steps" to "Recipe Steps"
5. Use `schedule.timeline` directly (all entries) — combine with any `prep_ahead_entries` for backwards compat with old session data
6. In `TimelineRow`, add an inline prep-ahead tag: if `entry.prep_ahead_window`, show a small tag like the existing resource badge
7. Update line 118: pass the full timeline to CookingGantt (no `is_prep_ahead` filter)

**File:** `frontend/src/components/session/CookingGantt.tsx`

**Commit:** `feat(03-01): unify frontend timeline — all steps in one view with inline tags`
